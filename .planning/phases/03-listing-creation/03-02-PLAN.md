**Prior decisions affecting this plan:**

- Phase 3 research already chose mobile-native capture with accepted-photo review, IndexedDB draft persistence, and OCR as advisory rather than auto-publish truth.
- The current stack already uses small focused helpers and route handlers, so upload and OCR work should follow the same split instead of being embedded in the page component.

**Concerns this plan must cover:**

- Missing OCR or storage credentials must fail gracefully into manual confirmation instead of blocking the seller from understanding what went wrong.
- Draft persistence has to be blob-safe, which means IndexedDB rather than localStorage.
</context>
