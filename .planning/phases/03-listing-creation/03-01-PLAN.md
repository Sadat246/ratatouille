**Prior decisions affecting this plan:**

- Phase 2 already guarantees a protected seller lane, so listing creation can assume authenticated business users and focus on business membership plus publish validation.
- Money values are stored in integer cents and lifecycle state is modeled with enums, so listing and auction writes should stay in that style instead of introducing floats or ad hoc status strings.
- The product rule is sealed, unopened goods only, and Phase 3 specifically needs three required photos plus a confirmed package date before publish.

**Concerns this plan must cover:**

- The publish path must reject auctions ending on or after the confirmed package date.
- The seller desk needs a real recent-listings query so the new Phase 3 UI is not built against static placeholders.
</context>
