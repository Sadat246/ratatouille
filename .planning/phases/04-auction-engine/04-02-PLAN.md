**Prior decisions affecting this plan:**

- Auctions must feel fast and live, but server time still decides winners, so the UI should refresh aggressively without pretending the browser clock is authoritative.
- Buyout stays a first-class sibling to bidding, not a hidden fallback action.
- The mock card-on-file gate belongs in Phase 4 so shoppers can conceptually qualify to bid before Stripe lands in Phase 6.

**Concerns this plan must cover:**

- The feed must only surface active, unexpired auctions while ended detail pages remain reachable and transparent about the final sale price or no-sale outcome.
- My Bids has to make winning vs outbid state obvious in one tap, because that is part of the core product feel for this phase.
</context>
