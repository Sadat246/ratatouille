**Prior decisions affecting this plan:**

- Proxy is only the first redirect layer; real authorization stays server-side in helpers and protected routes.
- Separate accounts per role is a product decision, so route enforcement should redirect rather than support in-app mode switching.
- The branded shells from Phase 1 stay intact; Phase 2 should make them session-aware instead of redesigning them.

**Deferred issues being addressed:**

- None beyond the active Phase 2 enforcement scope.

**Concerns being verified:**

- Consumers must never see business routes, businesses must never see consumer routes, and incomplete onboarding must never fall through into protected shells.
